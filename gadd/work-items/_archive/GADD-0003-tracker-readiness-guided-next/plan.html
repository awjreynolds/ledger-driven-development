Implementation Plan: Make GADD ready for external trackers and guided next actions

Work Item: GADD-0003
Status: approved
Source: plan.md

Context

This plan implements the approved PRD and SDD for GADD-0003. It addresses GitHub-first visibility, guided /gadd:next handoff, explicit /gadd:approve <work-item-id> approval for PRD and SDD gates, and PM shared-understanding guardrails.

Implementation Strategy

The work is implemented as command-surface and workflow-contract changes. The repo-local ledger remains canonical. GitHub is a managed projection surface, while Linear and Jira remain follow-on collaboration surfaces.

Slices

Slice 1: /gadd:approve command surface

Add a standalone approve skill, adapters, manifests, docs, and validation coverage for PRD and SDD approval only.

Slice 2: Guided /gadd:next and approval-routing gates

Update workflow skills so next_command and next_human_action are explicit, with PRD and SDD gates routing to /gadd:approve <work-item-id>.

Slice 3: GitHub-first tracker projection contracts

Document and validate GitHub issues for PRD/child work and GitHub PRs for SDD/plan/implementation review, with human confirmation and drift checks.

Slice 4: Shared-understanding guardrails and validation

Preserve bounded shared-understanding gates in PM skills and add validation so they are not accidentally removed.

Slice 5: End-to-end docs and package verification

Update user-facing docs and run package validation before verification and closure.
